# 第六章 集合与字典

## 一、集合及其表示

### 1. 概念

- 集合是成员（元素）的一个群集。集合中的成员可以是原子（单元素），也可以是集合
- 集合的成员必须**互不相同**
- 同一集合中所有成员具有**相同的数据类型**
- 集合成员一般是**无序的**

### 2. 位向量的实现方法

**存储结构：**

`unsigned short* bitVector`：一个`unsigned short`类型的一个数组，数组中的每个元素是一个16位的二进制数。

`bitVector[i]`的第`j`位可以是`0`或`1`，表示$i \times 16 + j$是否在集合之中。

**访问方式：**

确定在第几个元素里：`ad = x / 16​`

确定在元素的第几位：`id = x % 16`​

返回这个位的数字：`number = bitVector[ad] >> id % 2` 通过右移把这个数字移到个位上，然后对2取模

#### 部分关键函数的实现

**1. 判断元素是否存在**

```cpp
// 取集合元素x，判断元素x是否在集合中，如果是返回1，否则返回0
int getMember(const T x) const {
    if (x < 0 || x > setSize) return 0;
    int ad = x / 16;
    int id = x % 16;
    unsigned short elem = bitVector[ad];
    return ((elem >> id) % 2);
}
```

**2. 给指定元素设值**

```cpp
// 把值v送入集合元素x中
void putMember(const T x, int v) {
    if (x < 0 || x > setSize) return;
    int ad = x / 16;
    int id = x % 16;
    if (v == 1) {
        bitVector[ad] |= (1 << id);
    } else {
        bitVector[ad] &= ~(1 << id);
    }
}
```

**3. 判断是否R的子集**

```cpp
bool subSet(bitSet<T> &R){
    assert(setSize == R.setSize);
    for(int i = 0; i < vectorSize; i++){
        if(bitVector[i] & !R.bitVector[i])) return false;
    }
    return true;
}
```

### 3. 有序链表的实现方法

由于集合成员可以无限增加，因此用有序链表可以表示无穷全集合的子集。

## 二、并查集与等价类

### (一) 等价类

等价类是对象（或成员）的集合，在此集合中所有对象应满足等价关系（自反、对称、传递）

一个集合$S$中的所有对象可以通过等价关系划分为若干个互不相交的子集$S_1, S_2, S_3. \cdots$，它们的并就是$S$。这些子集称为**等价类**。

### (二) 并查集

建立等价类的方法是：

- 先把每个对象看作是一个单元素集合
- 然后按一定顺序将属于同一等价类的元素所在的**集合合并**

在此过程中将反复地使用一个**搜索**运算，确定一个元素在哪一个集合中

若两个等价对的元素不在同一个集合中，则可以进行**合并**

#### 并查集的三大操作

- `Union(Root1, Root2)`：合并操作

  ```cpp
  void Union(int Root1, int Root2){
      parent[Root1] += parent[Root2];		// 更新总个数
      parent[Root2] = Root1;		// 将Root2连接到Root1下面
  }
  ```

- `Find(x)`：查找操作

  ```cpp
  void Find(int x){
      if(parent[x] < 0) return x;
      else return parent[x] = Find(parent[x]);
  }
  ```

- `UFSet(s)`：构造函数

对于并查集来说，每个集合用一棵树表示

为此，采用树的双亲表示作为集合的存储表示。（第$i$个数组元素代表包含集合元素$i$的树节点。初始时，根节点的双亲为$-1$，用负数表示集合中的元素个数）

#### 不同的合并方法

如果不直接进行尾部合并的话，会增大树的深度，是搜索的时间复杂度增大，因此需要优化合并方式

1. **结点数多的树的根节点作根**

           parent[Root2] = Root1;

## 三、字典

字典是一些元素的集合，每个元素有一个称作**关键码**的域，不同元素的关键码互不相同。

在讨论字典抽象数据类型时，把字典定义为`<名字(key) - 属性(attribute)>`对的集合。

关于字典有以下**5种操作**：

- 确定**查找**一个指定的名字是否在字典中
- 搜索**(返回)**出该名字的对应属性
- **修改**该名字的属性
- **插入**一个新的名字及其属性
- **删除**一个名字及其属性

字典可以保存在线性序列$(e_1, e_2, \cdots)$中，$e_i$是字典中的元素，即**关键码**。

- 存储方式1：**有序顺序表**
  - 有序顺序表中关键码从左到右依次增大。
- 存储方式2：**有序链表**
  - 结点保存字典中的一个元素，各个节点按照结点中保存的数据值非递减链接。

#### 平均搜索长度ASL

设搜索第$i$个元素的概率为$p_i$，搜索到第$i$个元素所需要比较次数为$c_i$，则搜索成功的平均搜索长度：
$$
ASL_{succ} = \sum_{i = 1}^n p_i \cdot c_i. \quad (\sum_{i = 1}^n p_i = 1)
$$

#### 折半搜索算法性能判定树

若设$n = 2^h - 1$，则描述折半搜索的判定树是**高度为$h$的满二叉树**

对于位于第$i$层的节点，只需要搜索$i$次，因此
$$
ASL_{succ} = \log_2(n + 1) - 1
$$

## 四、散列

理想的搜索方法是不经过比较，一次直接从字典汇中得到搜索的元素。

建立一个元素存储位置`Address`与其关键码`key`之间的对应函数关系`Hash()`：
$$
Address = Hash(Key)
$$
在**插入时**按着函数计算存储位置，并按此位置存放。在**搜索时**对元素的关键码进行同样的计算，把求得的函数值当作元素存储位置，在结构中按此位置搜索。

按**散列方法**构造出来的表叫做散列表。所用转换函数叫做**散列函数**。
$$
hash(x) = x \% 73 + 13420
$$
但是存在不同的**关键码**，可能通过**散列函数**计算得到了相同的值，则称为**同义词**。

> 1. 给定一个关键码集合，需要选择一个**计算简单且地址分布均匀**的散列函数，避免或尽量减少冲突。
> 2. 拟定解决冲突的方案

### 散列函数的要求

1. 尽可能是简单的，能在短时间计算出结果
2. 定义域必须包括需要存储的全部关键码
3. 计算出来的地址尽可能均匀分布在整个地址空间中

#### 1. 直接地址法

$$
Hash(key) = a * key + b
$$

此类函数为一对一映射，一般不会产生冲突。但是要求散列地址与原关键码地址空间大小一致

#### 2. 数字分析法

设有$n$个$d$位数，每一位可能有$r$种不同的符号。这$r$种不同符号在各位上出现的概率不一定相同。根据散列表的大小，选取其中各种符号分布均匀的若干位作为散列地址。其中**分布均匀度**$\lambda_k$的公式：
$$
\lambda_k = \sum_{i = 1}^r (a_i^k - \frac{n}{r})^2
$$
如果$\lambda_k$值越小，表明在第$k$位各种符号分布得越均匀

然后可以取各关键码中分布均匀度小的几位作为记录的散列地址。

#### 3. 除留余数法

设散列表允许的地址数为$m$，取一个**不大于$m$但最接近于或等于$m$的质数**$p$作为除数，用以下函数把关键码转化为散列地址
$$
hash(key) = key \% p \quad p \leq m
$$
此时散列函数计算出的地址范围应该是$0 \sim p - 1$，从$[p, m - 1]$的地址实际上不能由散列函数计算出来，是用来处理冲突的。

#### 4. 平方取中法

计算构成关键码的标识符的内码的平方，然后按照取中间的若干位作为散列地址。

#### 5. 折叠法

此方法把关键码自左到右分成位数相等的几部分,  每一部分的位数应与散列表地址位数相同,  只有最后一部分的位数可以短一些。把这些部分的数据叠加起来，就可以得到具有该关键码的记录的散列地址。

两种叠加方法：

- 移位法：把各部分最后一位对齐相加
- 分界法：各部分不折断，沿各部分的分界来回折叠，然后对齐相加

>  相加后如果有进位都会直接删去。

### 处理冲突的闭散列方法

#### 1. 线性探查法

发生冲突就顺次向后寻找下一个空位置。
$$
ASL_{succ}
$$

$$
ASL_{unsucc}
$$

#### 2. 平方探查法

发生冲突就顺次向后寻找第$i^2$个位置

#### 处理冲突的开散列方法

设散列表地址空间的位置从$0 \sim  m - 1$，则关键码集合中的所有关键码被划分为$m$个子集，具有相同地址的关键码归于同一个子集。称为**同义词**。每一个子集称为一个**桶**。

每个位置有一个链表来存同义词。
